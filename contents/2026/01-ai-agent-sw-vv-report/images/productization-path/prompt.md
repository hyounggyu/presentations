# Productization Path

## Purpose

Show productization possibilities from the reference implementation without claiming it replaces ALM, eQMS, or RIM.

## Prompt

Create a 16:9 hand-drawn whiteboard style roadmap diagram. Use a two-row or gently curved path so the roadmap fits a normal 16:9 slide without becoming a panoramic strip. Start on the left with a box labeled "Reference Implementation". Draw a path through five product modules:

1. "Repository Template"
2. "SoT / Context Authoring"
3. "Regulatory Context Ingestion"
4. "Typst Document Generation"
5. "AI Review Assistant"

At the far right or lower-right, show a larger label: "Reasoning / Document Authoring Layer". Add small side labels outside the path: "ALM", "eQMS", "RIM" as systems it can connect to, not replace. Use dashed connecting lines, not takeover arrows. Add a tiny label near those dashed lines: "connects to".

Main message: the project can become a layer that structures evidence, generates documents, and assists review.

Style: hand-drawn whiteboard, business presentation diagram, muted blue/green/orange accents, simple icons for each module, spacious 16:9 layout, large readable labels.

## Negative Prompt

Do not show SWVNV replacing ALM, eQMS, or RIM. Do not use an ultra-wide panoramic strip layout. Do not include aggressive market-share imagery, sci-fi UI, dollar signs, "approved" stamps, long captions, paragraphs, or dense tiny text.
