# Current Implementation Structure

## Purpose

Summarize the current SWVNV implementation as one slide: SoT YAML, Context Registry, Typst documents, scripts, and agent skills.

## Prompt

Create a 16:9 hand-drawn whiteboard style architecture diagram for a presentation. Use a clean hub-and-flow layout with five labeled blocks.

Blocks:
- "SoT YAML" with small file cards: requirements, architecture, tests, risk controls, AI models
- "Context Registry" with file cards: MFDS guides, existing docs, review notes, working notes
- "Typst Documents" with document/page icons
- "Scripts" with a gear icon and labels: validate, trace, build
- "Agent Skills" with small tool cards: retrieval, drafting, review, PDF reader

Show SoT YAML flowing into Typst Documents. Show Context Registry flowing into Agent Skills. Show Scripts connected to Typst Documents and SoT YAML as validation/build automation. Keep the layout readable and explain the structure without long paragraphs.

Style: hand-drawn whiteboard, black marker, muted blue for SoT, green for Context, gray/orange for Scripts, light purple or neutral for Agent Skills.

## Negative Prompt

Do not include complex repository trees, tiny code, sci-fi dashboards, glowing UI, automatic approval, "Candidate", or "Final Document". Avoid long captions and dense labels.
